Things People Built

Show HN: SentrySearch — sub-second video search using Gemini's native video embeddings

Gemini can now natively embed video (not just frames or transcripts), and ssrajadh built a search tool on top of it. You point it at video footage and query it in natural language. The interesting bit is that the embeddings are over the actual video signal, not extracted text or keyframes. 400 points on HN, which for a Show HN is a lot. HN discussion.

Show HN: ProofShot — give AI coding agents eyes to verify the UI they build

Solves a real problem: coding agents generate UI code but can't see the result. ProofShot takes a screenshot after each change and feeds it back to the agent so it can catch its own visual bugs. Simple idea, useful if you're doing any agent-driven frontend work. HN discussion.

Show HN: AI Roundtable — let 200 models debate your question

From Opper AI. You submit a question and it gets routed to ~200 models, which then respond and debate each other. Feels like a fun way to see where models actually diverge on a topic. Probably more useful for comparison shopping models than for getting a definitive answer. HN discussion.

TurboQuant: extreme compression for LLMs

New from Google Research. TurboQuant pushes model quantization further than the usual 4-bit approaches. If you're running models on your own hardware and care about fitting bigger models into less VRAM, this is the kind of research that actually trickles down to your llama.cpp setup within a few months. 377 points on HN. Ensu: Local LLM app by Ente

Ente, the privacy-focused photo storage company, built a local LLM app called Ensu. Everything runs on-device. Ente has a good track record with privacy-first design, so this is a credible entry in the local-first AI space. HN discussion.

Hypura — storage-tier-aware LLM inference scheduler for Apple Silicon

If you're running LLMs on a Mac, Hypura is aware of Apple Silicon's unified memory architecture and schedules inference accordingly across storage tiers (RAM vs. SSD swap). 212 points on HN, which suggests people are actually finding it useful. HN discussion.

Developer Tools & Infrastructure

Auto mode for Claude Code

Simon Willison wrote up the new auto mode in Claude Code, which is Anthropic's answer to --dangerously-skip-permissions. Instead of blindly approving everything, a separate Claude Sonnet 4.6 classifier reviews each action before it runs. So you get the speed of auto-approve with a safety net. Whether that safety net is good enough is an open question, but it's a better design than a binary yes/no on all permissions.
Simon Willison

FastMCP

A framework for building MCP (Model Context Protocol) servers quickly. If you've been wanting to wire up your own tools to Claude or other MCP-compatible clients but found the boilerplate annoying, this is aimed at you. HN discussion.
Hacker News

Cloudflare: Sandboxing AI agents, 100x faster

Cloudflare announced dynamic Workers for sandboxing AI agent execution. The pitch: spin up an isolated environment for each agent action in single-digit milliseconds instead of seconds. Relevant if you're building anything where an agent needs to run untrusted code. HN discussion.
