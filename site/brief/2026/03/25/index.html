Sora is done

Goodbye to Sora

OpenAI's video generation product Sora is shutting down. Nearly 1,000 points on HN, which is a lot of attention for a product most people I know never found a real use for. The HN thread is mostly people saying they tried it once and forgot about it. Sora launched with incredible demos and then the actual product was... fine? The competition from Runway, Kling, and Veo moved fast. Hard to build a standalone business on video generation when it keeps getting bundled into everything else.

Supply chain attack on LiteLLM

Malicious litellm_init.pth in litellm 1.82.8 — credential stealer

LiteLLM v1.82.8 on PyPI was compromised with a credential stealer hidden in base64 in a .pth file. The nasty part: just installing the package triggers it, no import litellm needed. Version 1.82.7 had it too but required an import. If you're using LiteLLM, check your version now. Detailed writeup linked from Simon's post.
Simon Willison

Package Managers Need to Cool Down

Simon Willison used the LiteLLM incident to revisit the idea of dependency cooldowns: don't install updated packages until they've been out for a few days so the community can spot compromises. Turns out this is surprisingly well-supported across package managers already. Good practice that almost nobody follows.
Simon Willison

Claude Code gets auto mode

Auto mode for Claude Code

Anthropic added an auto mode to Claude Code as an alternative to --dangerously-skip-permissions. Instead of skipping permission checks entirely, a separate classifier model (Claude Sonnet 4.6) reviews each action before it runs. So you get the convenience of not clicking "yes" constantly, but with a safety net. Simon's writeup covers the details. This is the kind of incremental thing that actually matters for daily use.

Separate from their existing security bug bounty. This one targets AI-specific abuse patterns: prompt injection, agentic vulnerabilities, data exfiltration. Paying researchers to find these problems is straightforwardly good.
