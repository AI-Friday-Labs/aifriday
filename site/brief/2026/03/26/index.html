Thursday, March 26, 2026

Wild day in AI land. Sub-second video search just became possible thanks to Gemini's native video embedding, Google shipped Lyria 3 for music generation, and OpenAI launched a safety bug bounty program. Also: a supply chain attack hit LiteLLM this week — Simon digs into the numbers.

Things People Built

Sub-second video search with Gemini

Gemini can now natively embed video, and someone immediately built SentrySearch — a tool that searches through video content in under a second. No more scrubbing through footage manually. The repo includes a working demo you can try. HN discussion has people already thinking about use cases from security footage to lecture notes.
Hacker News

AI Roundtable: 200 models debate your question

Wild experiment from Opper: ask a question and watch 200 different AI models debate it in real-time. Not super practical, but fascinating to see how different models approach the same prompt. Great for understanding model behavior diversity. HN thread here.
Hacker News

Tools & Releases

Google Lyria 3: Music generation API

Google's newest music generation model is now available in paid preview through the Gemini API and AI Studio. You can test it today. They're also rolling out Lyria 3 Pro for creating longer tracks in production tools. Finally feels like music generation is getting real developer tooling.
Google AI Blog

Ensu: Local LLM app from Ente

The team behind Ente (encrypted photo storage) just shipped a local LLM app. Runs entirely on your machine, no cloud dependencies. Clean interface for chatting with models you download. Worth a look if you're tired of sending everything to APIs. HN discussion.
Hacker News

FastMCP: Framework for Model Context Protocol servers

New framework for building MCP servers (tools that connect AI models to external data/services). Aims to make it faster to ship context providers for Claude and other MCP-compatible models. Early but could be useful if you're building integrations. HN thread.
Hacker News

Claude Code gets 'auto mode'

Claude Code now has an auto mode where Claude makes permission decisions on your behalf with safety monitoring. Alternative to the --dangerously-skip-permissions flag. Simon breaks down how it works and the safeguards in place.
Simon Willison

Security & Policy

OpenAI launches Safety Bug Bounty

OpenAI's new bug bounty program focuses on AI-specific risks: agentic vulnerabilities, prompt injection, data exfiltration, and abuse patterns. If you've been poking at model safety, here's your chance to get paid for it. Separate from their existing security bug bounty.
OpenAI Blog

LiteLLM supply chain attack aftermath

The compromised LiteLLM package was live for 46 minutes. Using PyPI download data, researchers estimate 47,000 downloads during that window. Simon's analysis includes the credential-stealing mechanism and which packages were affected. If you use LiteLLM, worth checking. Related: his case for dependency cooldowns.
